WCAG

Text Alternatives

Definitions

Basically means that if there is an image, chart, video, or other objects that are not defined by text, they will need an alternative description (to be read out on screen readers).

All non-text content
A text description.
Control
Form Fields / User Inputs
These should have a name description.
Timebased Media
Video, Slides, Film or Audio
These should have a descriptive specification. A title is also a nice addition.
Live Timebased Media
Live: Stream, Television, Podcast, Radio
These should have an alternative text in a label.
Test
Test, Quiz, Exercise
If the element would be invalid when presented in text, these should have an alternative text description.
Sensory
Symphony performance, works of art, etc.
These should have an alternative text description.
CAPTCHA
User validation to confurm the content is accessed by a person rather than a computer.
These should have an alternative text description that identifies and describes the purpose, and alternative forms of CAPTCHA using output modes for different types of sensory perception to accommodate different disabilities.
Decoration
If the content is purely decorative, not visible, hidden or disabled.
These have to be able to be ignored by assisstive technology.

Adaptable Content

Create content that can be displayed in several ways, without losing information. Basically, good semantics and usage of titles and labels will achieve this. Also, there shouldn't be a restriction of the viewport, and content should be available in both portrait and landscape mode, and not require the user to switch between these orientations to access certain elements. Using the autofill tokens in input fields is also preferrable. Using ARIA landmarks to identify regions of a page.

Example: A required field in form is displayed with a red outline, an asterisk, and an icon with a label that reads "required".

Example: A website uses a multi-column lay-out, though in the HTML the content is provided in linear order, that is easily understandable for assisstive technology to navigate.

Distinguishable

Make it easier for users to see and hear content including separating foreground from background.

General guidelines are:

  1. Do not use color indicators for conveying information, indicating an action, prompting a response, or distinguishing a visual element.
  2. Do not auto-play audio, unless there is a mechanism is available to pause or stop the audio, or a mechanism is available to control audio volume independently from the overall system volume level.
  3. A contrast ratio of at least 4.5:1, except for large text (3:1), inactive or decorative items (none), logotype (none).
  4. An enhanced contrast ratio of at least 7:1, except for large text (4.5:1), inactive or decorative items (none), logotype (none).
  5. Text should be resizable, without loss of content. The usage of "rem" and percentages is helpful for this.
  6. Present text as images rather than images as text, and make this text fully customizable.
  7. Audio does not have background-noise unless this is less than 20 dB.
  8. Blocks of text must have colors that can be picked by user, a maximum width of 80 characters, text is not justified, line-spacing is at least 1.5 spaces, and paragraph spacing is at least 1.5 times bigger than the line-spacing, text is resizable up to 200% without needing to scroll horizontal.
  9. Images of text are purely decorative, or where the visual presentation is essential.
  10. Vertical scrolling should be a minimum width of 320 pixels, and horizontal scrolling a minimum height of 256 pixels.
  11. Non-text Contrast of at least 3:1.
  12. Text should have the following properties in CSS: line-height / line-spacing of at least 1.5 times the font-size, spacing following paragrahps at least 2 times the font-size, letter spacing of at least 0.12 times the font-size, word-spacing at least 0.16 times the font-size.
